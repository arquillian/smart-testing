=== Configuration


[cols="1,2,2,1", options="header"]
|===
|Property | Description |  Default | Possible Values

a|`smart.testing.mode`
|Set running mode
a|`selecting`
a|`ordering`, `selecting`

a|`smart.testing`
|Set strategies in CSV
| -
a|`new`, `changed`, `affected`, `failed`

a|`smart.testing.disable`
|Disable Smart Testing
a|`false`
a|`true`, `false`

a|`smart.testing.apply.to`
|Set plugin to apply Smart Testing in CSV
a|`surefire, failsafe`
a|`surefire`, `failsafe`
|===

=== Strategies

[cols="1,2,2,1", options="header"]
|===
|Property | Description | Default | Applicable Strategies

a|`scm.range.tail`
|Set first commit id for inspecting changes
a|`HEAD~0`
a|`new`, `changed`, `affected`

a|`scm.range.head`
|Set last commit id for inspecting changes
a|`HEAD`
a|`new`, `changed`, `affected`

a|`scm.last.changes`
|Set the number of commits from `HEAD` that you want to inspect
a|`0`
a|`new`, `changed`, `affected`

a|`smart.testing.affected.inclusions`
|Set classes to be included for scanning
a|
a|`affected`

a|`smart.testing.affected.exclusions`
|Set classes to be excluded for scanning
a|
a|`affected`

a|`smart.testing.affected.config`
|Set location of affected configuration file
a|
a|`affected`

|===

=== Getting insights of the execution

[cols="1,2,2", options="header"]
|===
|Property | Description | Default

a|`smart.testing.debug`
a|enables the debug mode (alternatively you can use Maven debug output `(-X)` flag)
a|`false`

a|`smart.testing.report.dir`
|directory where generated reports should be stored
a|`target` of the current module being built

a|`smart.testing.report.name`
|the file name of the generated report
a|`smart-testing-report.xml`

|===
